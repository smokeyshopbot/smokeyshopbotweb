How this works

The bot reads these settings from the database when showing checkout and wallet top-up options. If a method is disabled, users will not see that payment option. If a method is enabled but required details are empty, it is treated as disabled.

Disabled methods lock and hide their detail fields on this page. The saved values are kept so you can re-enable the method later without retyping everything.

Active payment sessions keep the exact wallet/payee details shown to that user, so a later update here will not make an already-created payment verify against a different wallet/payee.

Manual TxHash tolerance is read live from this page. It affects only user-submitted USDT TxHash auto-checks; automatic background USDT/Binance checks still require the exact unique amount.
